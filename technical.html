Introduction

WordPress is a popular CMS (content management system) platform used in over 60% of Websites worldwide. It's ever-growing popularity has deemed it the CMS of choice. Its simplicity offers a no-coding technique to Web Development.

WordPress operates on both front-end and back-end technologies.

  • WordPress front-end editing is very user-friendly. Users can navigate around very easily and edit, create, and maintain Webpages and blogs.
  • The back-end of WordPress consists primarily of PHP code. PHP (pre-hypertext processor) is a necessity to know when editing via the back-end. The back-end is where all of the foundations of WordPress take place. From FTP (file transfer protocol) to the cPanel, PHP is where it is at when it comes to WordPress back-end editing!
What you should already know

This guide assumes you have the following basic background:

  • A general understanding of WordPress and the World Wide Web.
  • Good working knowledge of both HTML and CSS.
  • Some Web development experience. If you are new to WordPress, try one of the tutorials on YouTube on WordPress.
Plugins

Plugins are additional software packages that are installed on WordPress to ensure additional features and operational techniques.

In contrast to a bare-boned WordPress platform, plugins give additional features and accesibilities.

It is higly important to update plugins as notified. Just wait a few days for a patch to become availabe, should there be security holes. Outdated plugins can serve as a gateway for users to enter malicous code into your WordPress site.

Custom CSS

CSS (cascade style sheets) coding can be used to customize themes. It is more useful to edit CSS in the cPanel or FTP, while creating a child theme. Entering direct CSS code into your WordPress site without a child theme will only result in the loss of code upon theme updates.

HTML

HTML is useful when editing in certain page editors. It is very useful to know HTML when tweaking pages to allow for custom programming..

W3Schools offers excellent educational platforms to learn code.

You can then proceed into your site's admin panel to custom code your site!

Themes

You find many themes in WordPress and there are many ways to work with them:

Select the desired theme and install it on a staging site first to see if it will make or break your site.

Update your theme first on a staging site to see how it will take effect.

Custom-code it by using a child theme.

Keep your theme updated to avoid unecessary problems down the road.

Media

Media can be uploaded using the "Media" tab in the admin panel.

Simply click "upload" and select the media type to be uploaded. If it is an image, always add alt text to ensure visibility for all users.

Always optimize your images either through a plugin, or photo editing software program.

Backups

Backups can be done with a plugin.

Get in the habit of backing up your WordPress site both weekly and before you make major changes to it. It is always great having a backup, should something go wrong in the process of making changes to your site.

Security

Security is so crucial to your site. Use security plugins to ensure optimal safety.

Malicious bots scan your site for vulnerabilities and then attack when the time is right. Be on top of this through prevention!

Dreaded White Screen

Occasionally, the white screen will present itself. This is what you can do to fix it:

  • If you cannot login, go to FTP and open up the configuration file. Add a few letters to the end of the name of the plugin file. This will deactivate all plugins.

    • Then log in to the admin panel.
    • Go through each plugin and activate each one to see where the problem occured.
    • If this fails, then check the database connection.
    • If this fails, check the current version of PHP and update as necessary.
Password protect your login pages

Use a plugin and configure it to password protect all of your login pages.

This will improve your site's safety, as it will prevent malicious bots from attempting brute-force attack methods to gain entry.

Staging sites

Staging sites are highly recommended when wanting to do changes to your site.

Install a staging plugin.

Perform all work on a staging site first to see its impact on the site.

Then, make necessary changes on your production site or push the changes through.

Responsivity

Make your site responsive through CSS coding.

Preferably, code the site on FTP or cPanel. This method is more effective and causes less confusion when you have multiple users on your site.
